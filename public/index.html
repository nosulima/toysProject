Toys Project Documentation

User routes

METHOD URL BODY DESCRIPTION
POST (register) /users { "fullName":"Michael Brown", "email": "michael.brown@example.com", "password": "brownpass", "role": "user" } Add a new user to the data base ,if you want to see the template of valid user.
POST (login) /users/login { "email": "noa@gmail.com", "password": "1234" } User login, if the email and password are correct the server will respond with a token, if you want to see the template of valid user.
PUT /users/:idEdit { "fullName":"Michael Brown", "email": "michael.brown@example.com", "password": "brownpass", "role": "user" } Update user details and save to databace.
DELETE /users/delete/:idDel none Delete a user from the data base.

Toys routes

METHOD URL BODY DESCRIPTION
GET /toys none Get a list of all the toys, will present 10 results per page.
GET /toys/search/s=Toy1 none search for a toy by searching follwing name or info after the "s=", it will present 10 results per page.
GET /toys/category/:catName none Get a list of all the toys in toys category, it will present 10 results per page.
POST /toys { "name": "Board Game: Monopoly Super", "info": "Classic family board game for hours of fun and strategic play.", "category": "Board Games", "img_url": "https://example.com/monopoly.jpg", "price": 60.00 } Add a new toy to the data base, You need a token for this query , if you want to see the template of valid toy
PUT /toys/:editid { "name": "Board Game: Monopoly Super", "info": "Classic family board game for hours of fun and strategic play.", "category": "Board Games", "img_url": "https://example.com/monopoly.jpg", "price": 60.00 } Update a toy in the data base,You need a token for this query.
DELETE /toys/:delid none Delete a toy from the data base, You need a token for this query.
